# Project Instructions

## Git Workflow
- Do NOT use git worktrees. Work directly in the main repository directory.
- Always create a feature branch from main before making code changes, then PR to main.
- Use descriptive branch names (e.g., `feature/add-sorting`, `fix/text-selection`).

## Development Process
- **ALWAYS show every feature on localhost:8080 and get user approval BEFORE pushing, creating PRs, or releasing.** Never skip the localhost preview step.
